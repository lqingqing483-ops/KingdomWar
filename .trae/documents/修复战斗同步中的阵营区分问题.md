# 修复战斗同步中的阵营区分问题

## 问题分析
当前战斗同步存在以下问题：
1. 战斗双方使用相同的战斗设置，都是蓝方
2. 双方使用同一个摄像机
3. 双方的放置区域没有区分

## 解决方案

### 1. 玩家阵营分配
- 修改 `NetworkManager.cs`，在玩家加入房间时自动分配阵营
- 使用现有的 `PhotonTeamsManager` 来管理玩家阵营
- 确保每个房间中玩家被分配到不同的阵营（红方和蓝方）

### 2. 摄像机设置
- 为每个阵营创建独立的摄像机
- 在场景中添加两个摄像机，分别对应红方和蓝方
- 根据玩家阵营激活对应的摄像机
- 使用 `CameraWork.cs` 来处理摄像机跟随

### 3. 放置区域区分
- 修改 `BattleManager.cs` 中的 `CanPlaceUnit` 和 `CanPlaceBuilding` 方法
- 根据玩家阵营限制放置区域
- 为红方和蓝方设置不同的放置边界

### 4. 战斗设置区分
- 修改 `BattleManager.cs`，根据玩家阵营加载不同的战斗设置
- 确保双方使用各自的出生区域和建筑布局

### 5. 网络同步调整
- 更新 `NetworkBattleManager.cs`，确保阵营信息在网络同步中正确传递
- 修改 `NetworkUnit.cs` 和 `NetworkBuilding.cs`，确保单位和建筑的阵营信息正确同步

## 实现步骤

1. **修改 NetworkManager.cs**：
   - 在 `OnPlayerEnteredRoom` 中为玩家分配阵营
   - 使用 `PhotonTeamsManager` 管理阵营分配

2. **修改 BattleManager.cs**：
   - 添加阵营相关的属性和方法
   - 修改放置区域检查逻辑
   - 添加摄像机管理逻辑

3. **场景设置**：
   - 在战斗场景中添加两个摄像机
   - 设置不同的放置区域边界

4. **测试和调试**：
   - 测试阵营分配是否正确
   - 测试摄像机切换是否正常
   - 测试放置区域限制是否生效
   - 测试网络同步是否正常

## 预期效果
- 战斗双方分别使用红方和蓝方的战斗设置
- 每个玩家只能在自己的放置区域内放置单位和建筑
- 每个玩家使用对应的摄像机视角
- 网络同步正常，阵营信息正确传递
