# 热更新方案说明

## 问题背景

打包后无法访问 `http://localhost/Win11/`，因为：
- Unity Editor 有内置 HTTP 托管服务
- 打包后的独立游戏没有 HTTP 服务器

## 三种热更新方案

### 方案一：纯本地资源（当前方案）✅

**适用场景**：开发测试、单机游戏

**配置**：
- `Build Remote Catalog`: false
- `Bundle Local Catalog`: true

**优点**：
- ✅ 无需 HTTP 服务器
- ✅ 打包即可用
- ✅ 稳定可靠

**缺点**：
- ❌ 无法动态更新 Addressables 资源
- ❌ 更新需要重新打包

**热更新能力**：
- ✅ 支持代码热更新（HotUpdate.dll）
- ❌ 不支持资源热更新

---

### 方案二：本地 + 远程混合模式（推荐）🔥

**适用场景**：正式项目、需要热更新

**核心思路**：
1. 首次启动使用本地资源
2. 检查远程版本文件
3. 有更新时下载到 PersistentDataPath
4. 后续从本地加载更新后的资源

**实现步骤**：

#### 1. 准备远程服务器

可以使用以下任意方式：
- **IIS/Apache/Nginx**：部署到本地或云服务器
- **Unity Addressables Hosting Service**：仅 Editor 使用
- **对象存储**：阿里云 OSS、AWS S3 等

#### 2. 修改配置

```csharp
// HotUpdateManager.cs
private const string BaseUrl = "http://你的服务器地址/Win11/";
private bool useRemoteServer = true; // 启用远程更新
```

#### 3. 版本检查流程

```
游戏启动
  ↓
检查远程 version.txt
  ↓
对比本地版本
  ↓
有更新？→ 下载更新包
  ↓
解压到 PersistentDataPath
  ↓
加载更新后的资源
```

#### 4. 目录结构

```
远程服务器：
http://你的服务器/Win11/
  ├── catalog_0.1.json      # Addressables catalog
  ├── catalog_0.1.hash
  ├── ui_mainmenu.bundle    # 资源包
  └── version.txt           # 版本文件

本地路径：
Application.persistentDataPath/
  └── HotUpdate/
      ├── HotUpdate.dll.bytes
      └── Addressables/     # 下载的更新资源
```

---

### 方案三：StreamingAssets + 远程下载

**适用场景**：大型资源更新

**流程**：
1. 首次打包将基础资源放入 StreamingAssets
2. 运行时检查远程更新
3. 增量下载更新资源到 PersistentDataPath

## 快速部署 HTTP 服务器（本地测试）

### 方法 1：使用 Python（当前使用）✅

**已配置**：
- 服务器地址：`http://localhost:8080/`
- 资源路径：`ServerData/StandaloneWindows64/`
- `useRemoteServer = true`

**启动命令**：
```bash
# 进入 ServerData 目录
cd d:\fs2025\unity\KingdomWar\ServerData\StandaloneWindows64

# 启动 HTTP 服务器
python -m http.server 8080

# 访问 http://localhost:8080/
```

**验证服务器**：
```bash
curl http://localhost:8080/catalog_0.1.json
```

**效果**：
✅ **与 Unity Editor 内置服务效果相同**
- 打包后的游戏可以访问资源
- Addressables catalog 正常加载
- 支持完整的热更新流程

**区别**：
- ❌ 需要手动启动（不会随 Unity 自动启动）
- ✅ 独立运行，关闭 Unity 后仍然可用
- ✅ 端口固定（8080），配置简单

**升级方案**：
