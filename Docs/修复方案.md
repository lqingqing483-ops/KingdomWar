# KingdomWar 修复方案

> 按"AI 辅助开发核心方法"编制：任务拆分 + 验收标准 + 独立分支 + 测试覆盖

---

## 任务 1：修复抽奖系统（LotterySystem）

**问题**：`LotterySystem.Draw()` 首行 `return new LotteryResult { success = false }`，`CanDraw()` 检查被注释，所有抽奖直接返回失败。

**影响文件**：
- `Assets/Scripts/HotUpdate/LotterySystem.cs`

**验收标准**：
- 调用 `Draw()` 时先检查 `CanDraw()`（免费次数或金币充足）
- 通过检查后执行加权随机抽卡
- 抽卡后正确扣除免费次数或金币
- 返回正确的 `LotteryResult`（success=true + card）

**分支**：`fix/lottery-system`

---

## 任务 2：修复 Addressables 阻塞调用

**问题**：`UIManager.cs` 和 `CardDatabase.cs` 使用 `WaitForCompletion()` 阻塞主线程。

**影响文件**：
- `Assets/Scripts/UI/UIManager.cs`
- `Assets/Scripts/Game/Cards/CardDatabase.cs`（如存在）

**验收标准**：
- 所有 `WaitForCompletion()` 替换为异步模式（回调或协程）
- 加载过程中不阻塞主线程
- 加载失败时有回退逻辑

**分支**：`fix/addressables-async`

---

## 任务 3：创建正式卡牌资产（CardData ScriptableObject）

**问题**：当前只有 4 张运行时创建的测试卡，没有正式卡牌资产，无法编辑和扩展。

**影响文件**：
- `Assets/Resources/Cards/`（新建目录）
- `Assets/Scripts/Editor/CardAssetCreator.cs`（修改或使用）
- `Assets/Scripts/Game/Cards/CardDatabase.cs`

**验收标准**：
- 创建 8 张正式 CardData 资产（骑士、弓箭手、火球、巨人、闪电、皮卡、圣水收集器、电击）
- 卡牌数据从 Resources 加载而非运行时创建
- 编辑器可编辑卡牌数据

**分支**：`feature/card-assets`

---

## 任务 4：添加音效系统

**问题**：整个项目没有任何 AudioSource 调用，无音效/BGM。

**影响文件**：
- `Assets/Scripts/Core/AudioManager.cs`（新建）

**验收标准**：
- 创建 AudioManager 单例
- 支持播放 BGM（循环）和 SFX（一次性）
- 支持音量控制（与 settingPanel 的滑块联动）
- 提供公共方法：`PlayBGM()`, `PlaySFX()`, `SetVolume()`

**分支**：`feature/audio-system`

---

## 任务 5：添加单位血条

**问题**：BuildingHealthBar 存在但单位没有血条。

**影响文件**：
- `Assets/Scripts/Game/Battle/Unit.cs`
- `Assets/Scripts/UI/UnitHealthBar.cs`（新建）

**验收标准**：
- 单位头顶显示血条（World Space UI）

**分支**：`feature/unit-healthbar`

## 实施策略

按方法论：
1. 每个任务建独立 Git 分支
2. 每个任务有明确的验收标准
3. 实施前先确认方案
4. 实施后验证验收标准
5. 合并回 master
