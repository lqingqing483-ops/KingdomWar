# ADR-002: 选择 HybridCLR 作为热更新方案

**日期**: 2026-07-05  
**状态**: 已实施  
**影响范围**: 热更新、代码加载、AssetBundle

## 背景

需要支持代码热更新以绕过 App Store/Android 审核周期，实现紧急 Bug 修复和活动内容推送。

## 选项

1. **HybridCLR** — 当前使用，解释执行 IL 元数据
2. **ILRuntime** — 第三方热更新框架，性能较差
3. **ToLua/Slua** — Lua 方案，需要额外维护 Lua 代码
4. **纯 Lua** — 全部逻辑 Lua 化，性能损失大

## 决策

选择 HybridCLR，理由：
- 代码用 C# 编写，无需学习 Lua
- 性能接近原生（比 ILRuntime 快 3-5 倍）
- Unity 2022.3 LTS 兼容性好
- 社区活跃，文档完善
- 与 Addressables 配合实现资源+代码双热更

## 后果

- ✅ C# 统一开发语言
- ✅ 热更新代码与主工程代码共享类型
- ✅ 性能接近原生
- ⚠️ 热更新代码需放在专门的程序集中
- ⚠️ 首次集成需要配置 AOT 泛型
