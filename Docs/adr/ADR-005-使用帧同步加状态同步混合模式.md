# ADR-005: 使用帧同步 + 状态同步混合模式

**日期**: 2026-07-05  
**状态**: 已实施  
**影响范围**: 战斗网络同步

## 背景

实时对战需要决定同步模式：帧同步（确定性）或状态同步（权威服务器）。

## 选项

1. **帧同步** — 同步操作指令，各端独立演算
2. **状态同步** — 服务器计算，客户端只显示结果
3. **混合模式** — 当前方案，帧同步为主 + 状态同步为辅

## 决策

选择混合模式，理由：
- 帧同步：30fps 固定频率同步操作（移动、攻击、部署）
- 状态同步：补充同步血量、状态变化等不确定性数据
- Photon PUN2 的 IPunObservable 用于状态同步
- FrameSyncManager + BattleSyncService 分别处理两类同步

## 后果

- ✅ 帧同步带宽小，只传操作指令
- ✅ 状态同步兜底，防止不同步
- ✅ 支持断线重连（全量状态同步）
- ⚠️ 帧同步要求各单位逻辑确定性
- ⚠️ 调试复杂（需要帧回放工具）
