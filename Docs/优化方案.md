## 1. 编码规范统一化

### 1.1 命名空间规范

**现状问题**：部分类有命名空间（`KingdomWar.Game.Battle`），部分类无命名空间（`UIManager`、`NetManager` 等），全局命名空间混乱。

**优化方案**：
```
KingdomWar.AI              → AIChatService, AIChatConfig
KingdomWar.Game.Battle     → BattleManager, Unit, Building, Spell, FrameSyncManager
KingdomWar.Game.Cards      → CardData, CardDatabase, CardEffectManager
KingdomWar.Game.Network    → NetworkManager, NetworkBattleManager
KingdomWar.Game.Pathfinding → GridManager, AStar, GridNode
KingdomWar.UI              → UIManager, basePanel, battlePanel, mainPanel
KingdomWar.Server          → NetManager, MsgBase, ServerRoot
KingdomWar.Tools           → Singleton<T>, ObjectPool, ObjectPoolManager
KingdomWar.HybridCLR       → LoadDll, ConsoleToScreen
KingdomWar.HotUpdate       → HotUpdateManager, LotterySystem, PlayerDataManager
KingdomWar.Editor          → CardAssetCreator, GridManagerEditor
```

### 1.2 代码风格对齐（参照 .NET 官方规范）

| 规范项 | 当前问题 | 建议修改 |
|--------|---------|---------|
| **花括号** | 部分类使用 Allman 风格，基本合格 | 保持统一 Allman 风格 |
| **缩进** | 混合使用 Tab 和空格 | 统一 4 空格 |
| **命名** | 变量命名不一致（`player1`、`enemyNamerText`） | 推荐 `playerOne`、`enemyNameText` |
| **public 字段** | 大量 public 字段 | 改为 `[SerializeField] private` + `public` 属性 |
| **var 使用** | 未系统使用 `var` | 在类型明确时使用 `var` |
| **文件作用域命名空间** | 使用 `namespace { }` | 改为 `namespace KingdomWar.Game.Battle;` |
| **XML 注释** | 缺少方法注释 | 为 public API 添加 XML 文档注释 |

### 1.3 引入 .editorconfig

在项目根目录创建 `.editorconfig`，统一缩进、命名规则、风格检查。

## 3. 性能优化

### 3.1 Addressables 阻塞调用

**问题**：`UIManager` 大量使用 `WaitForCompletion()` 阻塞主线程。

**优化**：全部改为异步模式，添加加载队列和超时处理。

```csharp
// ❌ 当前做法
var handle = Addressables.LoadAssetAsync<GameObject>(path);
handle.WaitForCompletion();

// ✅ 推荐做法
public async Task<GameObject> LoadPanelAsync(string path)
{
    var handle = Addressables.LoadAssetAsync<GameObject>(path);
    await handle.Task;
    return handle.Result;
}
```

### 3.2 对象池改进

**问题**：`ObjectPool` 出队后立即入队（`SpawnFromPool`），对象池实际上退化为循环队列，不支持扩容时对象可能丢失。

**优化**：
- 将 `Enqueue` 移到单独的 `ReturnToPool` 方法
- 支持运行时动态扩容
- 添加最大容量限制和溢出销毁策略

### 3.3 字符串拼接

**问题**：代码中多处使用 `+` 拼接字符串。

**优化**：使用 `$` 字符串插值或 `StringBuilder`。

### 3.4 协程与异步

**问题**：部分逻辑使用 `StartCoroutine`，混合使用 async/await。

**优化**：统一使用 async/await + UniTask（推荐），或统一使用 Coroutine。

### 3.5 Update 循环优化

**问题**：多个 MonoBehaviour 各自在 Update 中执行逻辑。

**优化**：引入 `GameLoopManager` 统一管理 Update 调度，支持暂停/恢复、帧率控制。

```csharp
public class GameLoopManager : MonoBehaviour
{
    public void Register(IUpdatable updatable, int priority = 0);
    public void Unregister(IUpdatable updatable);
    public void Pause();
    public void Resume();
}
```

## 4. 网络层优化

### 4.1 双网络栈问题

**问题**：同时维护自定义 TCP（NetManager）和 Photon PUN2（NetworkManager），职责不清晰。

**建议**：
- **短期**：明确区分职责 —— TCP 处理账号/数据服务，Photon 处理实时对战
- **长期**：考虑统一为一套网络方案（如全量迁移至 Photon 或自建服务器）

### 4.2 Thread.Abort 安全性

**问题**：`NetManager.Close()` 调用 `Thread.Abort()`（已过时）。

**优化**：改为 CancellationTokenSource 协作式取消。

### 4.3 忙等待 CPU 空转

**问题**：`MsgThread` 中空循环 `while (msgList.Count == 0) continue`。

**优化**：使用 `AutoResetEvent` 或 `ManualResetEventSlim` 信号通知。

### 4.4 场景名称不一致

**问题**：`NetworkManager` 加载 `"Main"`，`BattleManager.ExitBattle` 加载 `"mainScene"`。

**优化**：统一场景名称定义，使用常量或枚举。

```csharp
public static class SceneNames
{
    public const string MainMenu = "MainMenu";
    public const string Battle = "BattleScene";
    public const string Loading = "LoadingScene";
}
```

## 5. 安全性优化

### 5.1 BinaryFormatter 风险

**问题**：`DeckManager` 使用 `BinaryFormatter` 序列化卡组数据（`decks.dat`）。

**优化**：替换为 `Newtonsoft.Json` / `System.Text.Json` 或 Unity `JsonUtility`。

```csharp
// ✅ 推荐做法
public void SaveDecks(List<Deck> decks)
{
    string json = JsonUtility.ToJson(new DeckListWrapper { decks = decks });
    File.WriteAllText(savePath, json);
}
```

### 5.2 硬编码敏感信息

| 位置 | 内容 | 建议 |
|------|------|------|
| `AIChatConfig` | API Key 硬编码 | 移至加密配置文件或环境变量 |
| `NetManager` | `secret = "abc123"` | 从安全配置读取 |
| `HotUpdateManager` | `http://localhost:8080/` | 可配置化管理 |
| `PlayerDataManager` | `gold = 99799` | 移除调试代码 |
