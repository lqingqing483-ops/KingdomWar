# Spec: 宝箱系统

## Objective
实现宝箱系统，包括免费宝箱、胜利宝箱、皇冠宝箱，支持定时开启和加速开启。

## Tech Stack
- Unity 2022.3 LTS, C# 9.0
- UGUI + DOTween (开启动画)
- PlayerPrefs (宝箱状态持久化)
- Addressables (宝箱资源)

## Project Structure
```
Assets/Scripts/Game/Chest/
├── ChestManager.cs     → 宝箱核心逻辑
├── ChestData.cs        → 宝箱数据定义
└── ChestPanel.cs       → 宝箱UI面板
```

## Data Model

```csharp
public enum ChestType { Free, Victory, Crown, Special, Legendary }

[System.Serializable]
public class ChestData
{
    public ChestType chestType;
    public string chestName;
    public int goldReward;
    public int gemReward;
    public int cardCount;          // 产出卡牌数量
    public int[] cardRarityWeights; // 稀有度权重
    public int unlockTimeMinutes;  // 开启时间(分钟)
}

public class ChestSlot
{
    public ChestData chest;
    public long unlockStartTime;   // 开始解锁的时间戳
    public bool isUnlocking;
    public bool isUnlocked;
}

public class ChestManager : MonoBehaviour
{
    public static ChestManager Instance { get; private set; }
    public List<ChestSlot> slots;        // 4个槽位
    public ChestData GetFreeChest();      // 免费宝箱
    public void StartUnlock(int slotIndex);
    public void SpeedUpUnlock(int slotIndex); // 宝石加速
    public void OpenChest(int slotIndex);
}
```

## Code Style
- 命名空间: KingdomWar.Game.Chest
- 槽位数硬编码为 4（常量）
- 宝箱数据从 Resources 加载

## Testing Strategy
- ChestManager: 免费宝箱CD、宝箱开启计时、加速消耗宝石
- 4 槽位满时不能再获得宝箱
- 加速开启正确扣除宝石

## Boundaries
- Always: 槽位满时拒绝新宝箱、开启中不能重复开启、宝石不足不能加速
- Ask first: 新增宝箱类型、调整解锁时间
- Never: 直接修改 unlockStartTime

## Success Criteria
- 免费宝箱 4 小时 CD，CD 到后可领取
- 胜利后获得胜利宝箱（占用槽位）
- 宝箱可放入槽位开始计时解锁
- 可用宝石加速解锁
- 解锁后打开获得随机卡牌 + 金币
