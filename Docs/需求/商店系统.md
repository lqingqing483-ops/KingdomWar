# Spec: 商店系统

## Objective
实现游戏内商店系统，包括每日商店和特殊商店，支持金币/宝石购买卡牌和宝箱。

## Tech Stack
- Unity 2022.3 LTS, C# 9.0
- UGUI + DOTween (UI)
- PlayerPrefs (数据持久化)
- Addressables (资源加载)

## Project Structure
```
Assets/Scripts/Game/Shop/
├── ShopManager.cs       → 商店核心逻辑
├── ShopData.cs          → 商品数据定义
└── ShopPanel.cs         → 商店UI面板
```

## Data Model

```csharp
public enum CurrencyType { Gold, Gems }
public enum ShopType { Daily, Special }

[System.Serializable]
public class ShopItem
{
    public string itemId;
    public string cardName;      // 卡牌名（卡牌类商品）
    public CurrencyType currencyType;
    public int price;
    public int quantity;          // 购买获得数量
    public ShopItemType itemType; // Card, Chest, Gold, Gems
}

public class ShopManager : MonoBehaviour
{
    public static ShopManager Instance { get; private set; }
    public List<ShopItem> GetDailyItems();     // 每日商品
    public bool PurchaseItem(ShopItem item);   // 购买
    public void RefreshDailyShop();            // 刷新每日
}
```

## Commands
```
Compile: .\scripts\compile-check.ps1
Test: Unity Test Runner (EditMode)
```

## Code Style
- 命名空间: KingdomWar.Game.Shop
- 单例模式与已有 AudioManager 一致
- 所有耗时操作异步

## Testing Strategy
- ShopManager: 每日商品生成、购买扣费、刷新逻辑
- PlayerDataManager 联动：购买后金币/宝石正确扣除
- 边界：余额不足时购买失败

## Boundaries
- Always: 购买前校验余额、商品是否存在
- Ask first: 新增货币类型、商品类型
- Never: 直接操作 PlayerPrefs

## Success Criteria
- 每日商店每天刷新 3 张随机卡牌
- 可购买卡牌（金币）/ 宝箱（宝石）
- 购买后卡牌加入 PlayerDataManager
- UI 显示商品列表、价格、购买按钮
