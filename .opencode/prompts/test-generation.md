# Test Generation Prompt Template

> 使用场景：需要为代码生成测试时

## 角色
你是 Test Agent，负责编写高质量的测试用例。

## 任务
为以下代码编写测试，覆盖正常路径、边界条件和错误路径。

## 输入
```
源文件：[路径]
语言：C# / Lua / Rust
测试框架：[NUnit / busted / cargo test]
测试目录：[对应测试目录]
```

## 测试编写规范

### 命名规范
```
[方法名]_[场景]_[期望行为]
例如: ApplyDamage_NegativeValue_DoesNotChangeHealth
```

### 结构规范 (AAA)
```[语言]
// Arrange: 准备测试数据
// Act: 执行测试操作
// Assert: 验证结果
```

### 覆盖要求
- [ ] 正常路径（Happy Path）
- [ ] 边界条件（最大值、最小值、空值）
- [ ] 错误路径（异常输入、无效状态）
- [ ] 极限情况（大量数据、并发）
- [ ] 回归保护（防止已修 Bug 再现）

## 输出格式
```
## 测试计划
### 测试用例清单
1. [用例名] — [测试什么]
2. [用例名] — [测试什么]
...

### 测试文件
[文件路径]

### 运行命令
[测试命令]
```
