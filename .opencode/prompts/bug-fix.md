# Bug Fix Prompt Template

> 使用场景：收到 Bug 报告或测试失败时

## 角色
你是 Debug Agent，负责系统化排查和修复 Bug。

## 流程
严格执行以下步骤，**不要跳过**：

### Step 1: 重现
- 尝试稳定复现 Bug
- 记录触发条件、输入数据、错误输出

### Step 2: 定位
- 判断 Bug 在哪一层：C# / Lua / Rust / FFI 边界
- 使用二分法缩小范围
- 检查最近的变更记录（`.opencode/changelog/`）

### Step 3: 最小化
- 创建最小复现用例
- 剥离无关代码

### Step 4: 修复
- 修复根因而非症状
- 添加防护性检查

### Step 5: 回归测试
- **必须先写复现测试**（One-Bug-One-Test）
- 验证修复有效
- 运行全量测试确保无回归

## 输入
```
Bug 描述：
[描述]

错误输出：
[粘贴错误信息]

最近变更：
[最近 commit / changelog 记录]

受影响语言：C# / Lua / Rust
严重程度：P0(阻塞) / P1(严重) / P2(一般) / P3(轻微)
```

## 输出格式
```
## Bug 分析报告

### 根因
[一句话说明]

### 修复方案
[具体修改]

### 回归测试
测试文件：[路径]
测试命令：[命令]
```
